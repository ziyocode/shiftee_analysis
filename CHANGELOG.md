# 변경 이력 (Changelog)

이 프로젝트의 주요 변경 사항을 기록합니다.

## [1.0.4] - 2026-06-30

### 변경 (Changed)
- **신규 헤드리스 모드(`--headless=new`) 전환**: 구형 헤드리스는 UA에 'HeadlessChrome'이
  박혀 Shiftee 신규 `/app` SPA가 봇으로 감지해 부팅을 거부했다(브라우저 창을 띄워야만
  동작). 실제 Chrome과 거의 동일하게 동작하는 신규 헤드리스로 전환해, **창 없이 조용히
  실행하면서도** SPA가 정상 부팅하도록 함 (`login.py`).
- **HTML 리포트 디자인 현대화**: 신한(Shinhan) 블루 테마로 재디자인 — 그라데이션 hero,
  색상 코딩 요약 카드, 분포 바, status pill, zebra 테이블 (`html_report.py`).

### 제거 (Removed)
- GUI "브라우저 창 표시" 체크박스 제거 (v1.0.3에서 진단용으로 추가했던 것). 이제
  신규 헤드리스로 항상 창 없이 동작한다 (`gui_login.py`, `gui_main.py`).

## [1.0.3] - 2026-06-30

### 추가 (Added)
- **GUI "브라우저 창 표시" 체크박스**: 헤드리스에서 로그인 후 화면(SPA)이 뜨지 않을 때,
  체크하면 실제 브라우저 창을 띄워 진행 상황을 눈으로 확인할 수 있다. 보이는 브라우저는
  봇 감지에 잘 걸리지 않아 헤드리스 차단의 우회책도 된다 (`gui_login.py`, `gui_main.py`).

### 수정 (Fixed)
- **`.env` 불리언 파싱 깨짐**: Windows 메모장 등으로 만든 `.env`의 값 끝에 보이지 않는
  제어문자/잡문자가 붙어(`'false┘'` 등) `headless` 등 bool 설정 파싱이 실패하며 앱이
  크래시하던 문제 해결. 앞부분 true/false 토큰만 보고 관대하게 해석 (`settings.py`).

## [1.0.2] - 2026-06-30

### 수정 (Fixed)
- **리포트 다운로드 실패(Windows)**: 로그인 직후 신규 `/app` SPA의 사이드바(nav)가
  아직 렌더되기 전에 '리포트' 링크를 조회해, `text=리포트`를 60초 동안 못 찾고
  타임아웃되던 문제 해결. nav 링크가 DOM에 붙을 때까지 먼저 대기하고, 링크를 못
  찾으면 현재 URL의 회사 ID로 리포트 페이지 URL을 직접 구성하도록 변경
  (`attendance.py`).
- **원격 디버깅용 실패 아티팩트**: 다운로드 실패 시 `debug` 설정과 무관하게 항상
  스크린샷/HTML을 `logs/screenshots/`에 저장하도록 변경 (`.exe` 배포 환경 진단용).

## [1.0.1] - 2026-06-30

### 수정 (Fixed)
- **헤드리스 봇 감지 우회**: 신규 Shiftee `/app` SPA가 헤드리스(WSL 포함) 환경에서
  자동화를 감지하면 로딩 스피너에서 멈추던 문제 해결. 실제 데스크톱 Chrome처럼
  위장(User-Agent, `locale=ko-KR`, `timezone=Asia/Seoul`, 뷰포트, `navigator.webdriver`
  마스킹)하여 정상 부팅하도록 수정 (`login.py`).
- **실급여정산 다운로드 멈춤**: 모달의 "모두 선택"이 토글이고 직원이 진입 시 이미
  전체 선택돼 있어, 기존 코드가 한 번 클릭해 전체를 해제 → 직원 0명으로 다운로드
  버튼이 비활성인 채 720초 타임아웃으로 실패하던 문제 해결. 클릭 후 선택이 해제되면
  다시 눌러 전체 선택을 보장하도록 수정 (`attendance.py`).
