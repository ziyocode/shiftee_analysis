# Shiftee "위험" 직원 추출 기능 구현 계획

이 문서는 Shiftee 데이터를 다운로드하고, 엑셀 리포트의 "위험" 판단 로직을 Python으로 구현하여 대상자를 추출하는 방법에 대한 분석 및 계획입니다.

## 1. 개요

- **목표**: `shiftee_data1.xlsx` (Realtime)와 `shiftee_data2.xlsx` (Payroll) 데이터를 분석하여, 초과근로 "위험" 상태인 직원을 자동으로 식별.
- **참고 로직**: `레포트_20251101-1130_뱅킹인프라본부.xlsx` 파일의 `계산` 시트 로직.

## 2. 데이터 흐름

1.  **데이터 다운로드**: 기존 `shiftee_analysis.py`의 다운로드 기능을 사용하여 최신 데이터 확보.
2.  **데이터 로딩**: 확보된 두 개의 엑셀 파일을 Pandas DataFrame으로 로드.
3.  **지표 계산**: 엑셀 수식을 Python 로직으로 변환하여 주요 지표 계산.
4.  **위험군 필터링**: 계산된 지표를 기반으로 위험군 추출.

## 3. 상세 로직 분석 (`계산` 시트 기준)

"위험" 여부는 `계산` 시트의 **U열 (적정성)**에서 판단합니다.

### 3.1. 핵심 수식
- **적정성 (U열)**: `IF(O > Q, "위험", "정상")`
    - **O열 (실제 초과 근로시간)**: `실제 초과 근로시간(조기출근제외)`
    - **Q열 (법정 초과 한도)**: `DAY(기준일)/7 * 12` (월 흐름에 따른 누적 한도 추정치)

### 3.2. 변수별 계산 방법

| 변수 (엑셀 열) | 설명 | 데이터 원천 / 로직 | 비고 |
| :--- | :--- | :--- | :--- |
| **B (직원)** | 직원명 | `shiftee데이타!B` (Realtime) | |
| **D (소정근로)** | 소정근로시간 | `shiftee데이타!G` (Realtime) | |
| **K (유급휴가)** | 유급휴가시간 | `shiftee데이타!L` (Realtime) | |
| **L (법정근로)** | 법정근로시간 | `D - K` | 법정 의무 근로시간 |
| **G (실제근로_보정)** | 실제근로(보정) | `SUMPRODUCT(...)` from `shiftee데이타2` (Payroll) + 보정값 | **가장 복잡한 로직**. Payroll 데이터의 상세 타임스탬프 기반 계산 필요. |
| **H (실제근로_단순)** | 실제근로(단순) | `shiftee데이타!J` (Realtime) + 보정 | |
| **N (실제초과)** | 실제초과근로 | `IF((G-L)<0, 0, G-L)` | 보정된 실제근로 기준 초과 시간 |
| **O (실제초과_최종)** | 판정용 초과근로 | `IF(조건, N, H-L)` 등 복합 로직 | 조기출근 제외 등 예외처리 포함 |

### 3.3. 구현 시 주의사항 (G열 로직)
`G열`은 `shiftee데이타2` (Payroll) 시트의 데이터를 참조하여 정밀한 근로시간을 재계산합니다.
- **엑셀 수식**: `SUMPRODUCT((Payroll!Name=Name) * ((EndTime - StartTime) - BreakTime))` 형태
- **Python 구현**:
    - `shiftee_data2.xlsx`를 로드하여 사원별로 GroupBy.
    - 각 근무 기록의 `(퇴근시간 - 출근시간) - 휴게시간` 합산.
    - 이를 `shiftee_data1.xlsx`의 사원 정보와 병합(`Merge`).
